# 配置环境变量

环境变量(Environment Variables)是操作系统或程序运行环境中存储的键值对(key-value)数据，用于配置系统和应用程序的行为。

例如：

环境变量 作用
PATH 可执行文件的搜索路径
C_INCLUDE_PATH GCC 搜索 C 语言头文件的路径
CPLUS_INCLUDE_PATH GCC 搜索 C++ 头文件的路径
LIBRARY_PATH GCC 搜索库文件的路径
LD_LIBRARY_PATH Linux 运行时加载动态库的路径

通过 echo 命令可以查看环境变量的值：

echo $PATH      # 查看 PATH 环境变量的值

临时设置环境变量可以使用 export 命令：

export 变量名=变量值

例如：

export PATH="$PATH:路径"      # 向 PATH 环境变量追加路径，使用冒号分隔

# 用户级配置

持久化设置用户级环境变量通常通配置 ~/.bashrc 文件实现：

# .bashrc 原本的内容
# ....

# 通过 export 配置环境变量
export 变量名=变量值
export 变量名=变量值
export 变量名=变量值

修改后不会立刻生效，需要重启终端，或者通过 source ~/.bashrc 加载配置。

# 系统级配置

持久化设置系统级环境变量通常在 /etc/profile.d/ 目录中添加配置文件实现：

# 通过 export 配置环境变量
export 变量名=变量值
export 变量名=变量值
export 变量名=变量值

修改后不会立刻生效，需要重启终端，或者通过 source /etc/profile.d/xxx.sh 加载配置。

也可以通过编辑 /etc/environment 文件配置系统级环境变量，该配置文件中直接写键值对，而不使用 export：

变量名=变量值
变量名=变量值
变量名=变量值
/etc/environment 由 PAM 管理，如果用户不使用 PAM，则此配置不会生效。

# sudo

通过 sudo 执行命令时，环境变量会被重置为 root 用户的配置，可以使用 -E 选项保留当前用户的环境变量：

sudo -E command                         # 保留全部环境变量
sudo --preserve-env=VAR1,VAR2 command   # 保留指定变量

也可以修改配置 /etc/sudoers 保留用户环境变量：

# 此行表示重置环境变量，改为 !env_reset 即可不重置环境变量
Defaults        env_reset

或者针对性的保留环境变量：

# 保留所有环境变量（安全风险较高）
Defaults env_keep += "*"

# 保留特定环境变量（推荐）
Defaults env_keep += "PATH HOME JAVA_HOME"
